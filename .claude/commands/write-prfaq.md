# Write PR/FAQ

Write a PR/FAQ document for the given feature or product idea.

## Instructions

1. Read the PR/FAQ template at `docs/templates/PRFAQ.md`
2. Ask the user for the feature/product name if not provided as an argument: $ARGUMENTS
3. Use the **product-strategist** agent to think through the customer problem, solution, and competitive positioning
4. Fill in every section of the template — do not leave any section with placeholder text
5. The Press Release section should be written in customer-facing language (no jargon)
6. The FAQ section should address real objections and concerns
7. Save the completed document to `docs/prfaq/PRFAQ-<feature-name>.md`

## Output

Produce a complete PR/FAQ document. After writing, summarize the key decisions and highlight any sections that need stakeholder input.
