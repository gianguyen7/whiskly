# Draft Architecture

Create an Architecture Decision Record (ADR) for the given technical decision.

## Instructions

1. Read the ADR template at `docs/templates/ADR.md`
2. The decision to document: $ARGUMENTS
3. Use the **technical-architect** agent to:
   - Analyze the context and constraints
   - Identify at least 3 options with pros/cons
   - Make a clear recommendation with rationale
4. Consider the existing project structure in `src/` and `infra/`
5. Include estimated effort and operational cost for each option
6. Document risks and mitigations
7. Save the completed ADR to `docs/adr/ADR-NNN-<decision-title>.md` (auto-increment the number)

## Output

Produce a complete ADR following the template. End with a clear recommendation and the follow-up actions needed to implement it.
