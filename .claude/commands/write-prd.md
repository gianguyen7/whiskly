# Write PRD

Write a Product Requirements Document for the given feature.

## Instructions

1. Read the PRD template at `docs/templates/PRD.md`
2. The feature to document: $ARGUMENTS
3. Use the **product-strategist** agent to define the problem, personas, and success metrics
4. Use the **technical-architect** agent to fill in technical considerations
5. Fill in every section of the template with specific, actionable content
6. Requirements should be testable — avoid vague language like "fast" or "user-friendly"
7. Include concrete success metrics with measurement methods
8. Save the completed document to `docs/prd/PRD-<feature-name>.md`

## Output

Produce a complete PRD. Flag any open questions that need resolution before engineering work begins.
